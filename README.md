# TS-PROJ-2
Название: Приложение поиска информации о разработчике на GitHub  
Стек: TypeScript, React, HTML5, CSS-модули

Задачи: 
1. Реализовал строго типизированное react-приложение, которое получает данные о разработчике из публичного API GitHub.
2. Создал разные типы данных для сущностей API (GithubUser/GithubError) и для внутреннего интерфейса приложения (LocalGithubUser) для изоляции приложения от нестабильности внешних API.
3. Спроектировал гибкую структуру UI-компонентов с использованием служебных типов TypeScript (Pick) и декларативного рендеринга списков на основе конфигурационных массивов.
4. Реализовал отправку асинхронного запроса к конечной точке GitHub REST API, валидацию ответов GitHub API в рантайме с помощью кастомного Type Guard, базовую обработку ошибок (если пользователь не найден, стейт сбрасывается в null) и добавил валидацию на пустые строки.
5. Реализовал маппинг данных: написал утилиту-трансформатор для преобразования объектов из GitHub API во внутренний интерфейс приложения. Это позволило очистить данные от 20+ избыточных полей, нормализовать свойства к единому формату и изолировать UI-компоненты от структуры бэкенда.
6. Реализовал компонентную архитектуру стилей с помощью CSS-модулей, что обеспечило полную изоляцию стилей компонентов и исключило конфликты имён классов.

Фичи:
1. Добавил переключатель дневной и ночной темы: при нажатии на кнопку сайт мгновенно меняет цветовую гамму, а текст и иконка автоматически подсказывают, какой режим включится следующим (Луна/Солнце и Dark/Light).
2. После успешного нажатия кнопка поиска сама очищает поле ввода, избавляя человека от необходимости вручную стирать старый текст для нового поиска.
3. Если пользователь введет имя несуществующего аккаунта, сайт не «зависнет» и не выдаст системную ошибку на английском. Вместо этого интерфейс отобразит лаконичную и понятную подсказку «No result» слева от кнопки поиска.
